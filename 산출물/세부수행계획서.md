# 마타에듀 POC add on 태스크 세부 수행 계획서

### 관련 문서
이니셜 미팅 회의록
경쟁사 분석 자료

## 개요

* 핵심 목표 : 2개의 별도 워크플로우에 대한 구현
    1. 개별 문항 분석 : 문항 1개를 업로드하여 메타데이터 분석 결과를 확인
    2. 기출 문제 분석 : 시험지 1개를 업로드하여 포함된 모든 문항의 통계 결과를 확인
* poc 기간 : 총 6일 (10/31, 11/5, 11/6, 11/7, 11/12, 11/13)
* 마감일 : 11/13
* 핵심 기술 스택 : streamlit, gemini, paddlepaddle, postgresql

## POC 상세 범위

워크플로우 A. 핵심 공유 엔진 과 B,C 사용자 워크플로우로 구성

### 워크플로우 A : 핵심 공유 엔진
* 범위 : 두 워크플로우가 공통적으로 호출하는 핵심 모듈
* 주요 산출물
    - 데이터 표준 정의 : ai가 반환할 메타데이터의 구조
    - ocr 연동 모듈 : pdf/이미지 파일을 입력받아 텍스트를 반환
    - 메타데이터 분석 파이프라인 : 문항 텍스트 1개를 입력받아 정의한 메타데이터 구조의 json 반환

### 워크플로우 B : 개별 문항 분석기
* 범위 : 문항 1개를 즉시 분석하는 streamlit 페이지
* 주요 산출물
    * 개별 문항 입력 ui : 사용자가 개별 문항을 이미지로 업로드
    * 즉시 분석 로직 : 워크플로우 A 호출
    * 분석 결과 표시 ui

### 워크플로우 C : 기출 문제 분석기 
* 범위 : 시험지 1개를 통계 분석하는 streamlit 페이지
* 주요 산출물
    * 문항 분리 로직
    * 엔드 투 엔드 처리 로직
        * 시험지 업로드 -> ocr, 파싱 -> 개별 문항 별로 워크플로우 A 호출 -> 모든 결과를 db 저장
        * 시험지에 대한 통계 결과 반환
    * 분석 리포트 ui 
        * db에 저장된 데이터를 집계하여 시각화
    * 데이터베이스 모델

## 수행 범위
### 성공 기준
- 사용자가 ui에서 시험지 파일을 업로드합니다
- 사용자가 ui에서 개별 문항을 업로드합니다.
- 워크플로우가 자동으로 실행되어 문항별 분석 결과가 db에 저장됩니다
- 사용자는 분석 결과 화면에서 단원별 / 난이도별 문항 분포 차트를 확인할 수 있습니다.
- 사용자는 문항 분석 결과 화면에서 출제 의도, 난이도 등 메타 데이터를 확인할 수 있습니다.

### 범위 외 (명시적으로 제외하는 기능)
- 로그인 및 사용자 인증 기능
- 다양한 시험지 양식에 대한 100% 완벽한 파싱 및 예외처리
- 대규모 트래픽 처리를 위한 성능 최적화(latency에 대한 제약 고려하지 않음)

### 핵심 리스크 및 가정
- gemini의 출제 의도, 난이도 등에 대한 할루시네이션은 검증할 수 없습니다.
- gemini의 성능이 기대 이하일 경우, mock up 데이터로 임시 대체하여 플로우의 작동을 우선적으로 검증합니다. 
- 완벽한 ocr 성능을 구현할 수 없습니다. (복잡한 수식, 이미지, 표 등)
- 파싱 성능을 100% 구현하지 않습니다.
